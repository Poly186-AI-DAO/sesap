# Overview

The User flow document outlines the steps a user will take to utilize the features of the SESAP MVP platform. The overarching goal of SESAP is to be the platform for self-executing social agreements, and each phase of the MVP roadmap works towards that goal in different ways.  

In Phase 1, Simple Agreement Creation, the platform allows users to create agreements quickly and easily through a user-friendly interface. The addition of the e-signature functionality and the VERTEX VISION API for driver's license uploads further streamlines the process and adds an element of trust.  

Phase 2, Contract Minting and Analysis, builds upon this by incorporating the analysis of uploaded agreements using DOCUMENT AI to help users understand the terms and conditions. The agreements are then minted and stored on the blockchain, making them secure and easily accessible for future reference.  

Phase 3, Organization Management, expands the scope of the platform by allowing users to create and manage entire organizations. Users can define and generate job postings for any vacant positions, making it a one-stop-shop for all their organization's needs.  

Phase 4, Cloud Talent Powered Features, utilizes the power of GPT-3 and Codex APIs to test applicants' understanding of organizations or projects, ensuring the right fit for the organization.  

Phase 5, DocAI Warehouse for Document Management, provides a centralized location for document management and allows for organization, project, or product discovery to increase conversion rates.  

Finally, in Phase 6, Improved User Experience, the platform incorporates translation and custom recommendations to enhance the user experience. The discovery engine allows for cross-industry discovery solutions, making it even easier for users to find the right fit for their needs. The improved UI also makes monitoring agreements, contracts, and organizations a seamless process.  

Overall, the User flow document details the steps a user will take to utilize the features of the SESAP MVP, each step working towards the goal of being the platform for self-executing social agreements.

# User Flow

## Phase 1:

### Simple Agreement Creation

1.  User accesses the SESAP platform and selects the agreement creation option  
    
2.  User inputs the necessary information for the agreement  
    
3.  User uploads a driver's license using the VERTEX VISION API for trustability and easy contract creation  
    
4.  User signs the agreement electronically using the e-signature functionality  
    
5.  The agreement is saved and stored on the platform for future monitoring  
    

## Phase 2:

### Contract Minting and Analysis

1.  User accesses the SESAP platform and selects the agreement creation option  
    
2.  User selects from customizable contract templates or inputs the necessary information for the agreement  
    
3.  User uploads the agreement as a PDF for analysis using DOCUMENT AI  
    
4.  The agreement is analyzed to help the user understand the terms and conditions  
    
5.  User signs the agreement electronically using the e-signature functionality  
    
6.  The agreement is minted and stored on the blockchain for future reference  
    
7.  The user is able to monitor the agreement status on the platform

## Phase 3:

### Organization Management

1.  User accesses the SESAP platform and selects the option to create a company or organization  
    
2.  User inputs the necessary information for the company or organization structure  
    
3.  User defines and generates job postings for any vacant positions within the organization  
    
4.  User is able to monitor the status of the company or organization on the platform

## Phase 4:

### Cloud Talent Powered Features

1.  User accesses the SESAP platform and selects the option to manage an organization  
    
2.  User inputs the necessary information for the job posting  
    
3.  User tests the applicant's understanding of the organization or project using GPT-3 and Codex APIs  
    
4.  User is able to monitor the status of the organization or project on the platform

## Phase 5:

### DocAI Warehouse for Document Management

1.  User accesses the SESAP platform and selects the option to manage documents  
    
2.  User inputs the necessary information for the document and selects the type of document  
    
3.  The document is stored in the DocAI Warehouse for future reference  
    
4.  User is able to access the document on the platform for review and updates  
    
5.  User is able to experience organization, project, or product discovery for increased conversion rates across digital properties

## Phase 6:

### Improved User Experience

1.  User accesses the SESAP platform and selects the option to view an agreement or contract  
    
2.  The agreement or contract is automatically translated into the user's preferred language  
    
3.  The user receives custom recommendations for contracts, agreements, organizations, projects, and jobs based on the user's preferences and history  
    
4.  User is able to access the discovery engine for cross-industry discovery solutions  
    
5.  Users are able to monitor the status of agreements, contracts, and organizations on the platform with an improved UI.
