# Overview

The user story document outlines the general user stories for the SESAP MVP (Minimum Viable Product). It details the various features and functionality that customers can expect at each phase of the MVP.  

**Phase 1:** Simple Agreement Creation, the user will be able to create an agreement on the SESAP platform and sign it electronically using the e-signature functionality. The user will also be able to upload their driver's license for trustability and easier agreement creation, and monitor the agreement status on the platform.  

**Phase 2:** Contract Minting and Analysis, the user will have access to customizable contract templates and be able to upload their agreement as a PDF for analysis. The agreement will then be minted and stored on the blockchain for future reference. The user will also be able to monitor the agreement status on the platform.  

**Phase 3:** Organization Management, the user will be able to create a company or organization on the SESAP platform and define and generate job postings for any vacant positions within the organization. The user will also be able to monitor the status of the company or organization on the platform.  

**Phase 4:** Cloud Talent Powered Features, the user will be able to manage an organization using cloud talent powered features and test the applicant's understanding of the organization or project using GPT-3 and Codex APIs. The user will also be able to monitor the status of the organization or project on the platform.  

**Phase 5:** DocAI Warehouse for Document Management, the user will be able to manage their documents on the SESAP platform and store them in the DocAI Warehouse for future reference. The user will also be able to experience organization, project, or product discovery for increased conversion rates across digital properties.  

**Phase 6:** Improved User Experience, the user will be able to view agreements or contracts in their preferred language, receive custom recommendations for contracts, agreements, organizations, projects, and jobs based on their preferences and history, and access the discovery engine for cross-industry discovery solutions.  

Overall, the user story document outlines the step-by-step process of what customers can expect when using the SESAP MVP. It highlights the key features and functionality that will be available at each stage, and provides a roadmap for what customers can expect as the MVP evolves and expands over time.

# User Stories

## Phase 1:

### Simple Agreement Creation

1.  As a user, I want to be able to create an agreement on the SESAP platform, so I can easily manage my legal agreements.  
    
2.  As a user, I want to have the ability to upload my driver's license for trustability and easier agreement creation, so I don't have to worry about the agreement's validity.  
    
3.  As a user, I want to be able to sign the agreement electronically using the e-signature functionality, so I can save time and eliminate the need for physical signatures.  
    
4.  As a user, I want to be able to monitor the agreement status on the platform, so I can stay up-to-date with any changes or updates.  
    

## Phase 2:

### Contract Minting and Analysis

1.  As a user, I want to have access to customizable contract templates, so I can easily create agreements that fit my specific needs.  
    
2.  As a user, I want to have the ability to upload my agreement as a PDF for analysis, so I can understand the terms and conditions more effectively.  
    
3.  As a user, I want the agreement to be minted and stored on the blockchain for future reference, so I have a secure and immutable record of the agreement.  
    
4.  As a user, I want to be able to monitor the agreement status on the platform, so I can stay up-to-date with any changes or updates.  
    

## Phase 3:

### Organization Management

1.  As a user, I want to be able to create a company or organization on the SESAP platform, so I can easily manage my business entities.  
    
2.  As a user, I want to be able to define and generate job postings for any vacant positions within the organization, so I can attract the right talent.  
    
3.  As a user, I want to be able to monitor the status of the company or organization on the platform, so I can stay informed about its progress.  
    

## Phase 4:

### Cloud Talent Powered Features

1.  As a user, I want to be able to manage an organization using cloud talent powered features, so I can maximize efficiency and productivity.  
    
2.  As a user, I want to be able to test the applicant's understanding of the organization or project using GPT-3 and Codex APIs, so I can ensure a good fit.  
    
3.  As a user, I want to be able to monitor the status of the organization or project on the platform, so I can stay informed about its progress.  
    

## Phase 5:

### DocAI Warehouse for Document Management

1.  As a user, I want to be able to manage my documents on the SESAP platform, so I can easily access and review them.  
    
2.  As a user, I want to be able to store my documents in the DocAI Warehouse for future reference, so I have a secure and easily accessible place for them.  
    
3.  As a user, I want to be able to experience organization, project, or product discovery for increased conversion rates across digital properties, so I can make informed decisions.  
    

## Phase 6:

### Improved User Experience

1.  As a user, I want to be able to view agreements or contracts in my preferred language, so I can understand them more easily.  
    
2.  As a user, I want to receive custom recommendations for contracts, agreements, organizations, projects, and jobs based on my preferences and history, so I can find what I'm looking for more efficiently.  
    
3.  As a user, I want to be able to access the discovery engine for cross-industry discovery solutions, so I can explore new opportunities.  
    
4.  As a user, I want to be able to monitor the status of agreements, contracts, organizations, projects, and jobs from a single dashboard, so I can have a clear overview of everything in one place.
